github 연동하는 법

유형git
링크https://webstoryboy.co.kr/1966

1.프로젝트 저장소를 github에 만든 후 생성된 로컬 git 저장소를 복사해둔다 .

2. VSCODE 터미널에 작업한다.

git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/oLangS2/OHTUBE.git //생성된 로컬 git 주소
git push -u origin main
git add . //모든 파일 올리기

3.GIT 상태 확인하기

git status

새로운 파일이나 수정된 파일을 올릴지 말지 상태를 확인할 수 있습니다.

4. git 수정 후 올리기

git add .
git status

위에 처럼 했을 때 에러가 났는데

“warning: in the working copy of 'package.json', LF will be replaced by CRLF the next time Git touches it”

경고 : bora.txt에서 LF는CRLF로 대체됩니다. 파일은 작업 디렉토리에 원래 줄 끝이 있습니다.

이렇게 플랫폼(OS)마다 줄바꿈을 바라보는 문자열이 다르기에 형상관리를 해주는 Git이 바라볼 땐 둘 중 어느 쪽을 선택할지 몰라 경고 메세지를 띄워준 것.

해결방안(window. DOS)

git config --global core.autocrlf true

해결방안(MAC)

git config --global core.autocrlf input

git commit -m "CSS 설정완료"
git push -u origin main

차례대로 터미널에 입력 후 깃허브 확인해 보면 잘 올라가있는 모습을 볼 수 있다.
